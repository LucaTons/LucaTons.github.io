Le applicazioni Web

Quest'anno ho studiato come vengono sviluppate le applicazioni web: dai database al PHP, fino alla collaborazione in gruppo su un progetto reale. Il percorso ci ha portato a costruire un backend completo con operazioni CRUD su più tabelle, usando PHP come linguaggio lato server e MySQL per la gestione dei dati.

Abbiamo strutturato il progetto separando la logica applicativa dalle query SQL attraverso un livello di astrazione sul database: invece di scrivere query direttamente nei file che gestiscono le richieste HTTP, abbiamo creato classi dedicate che incapsulano tutte le operazioni sui dati. Questo rende il codice più ordinato, più facile da testare e più sicuro.

Cybersecurity applicata al web

La parte che mi ha colpito di più è stata la cybersecurity applicata al web. Ho analizzato e testato vulnerabilità reali:

SQL Injection: si verifica quando l'input dell'utente viene inserito direttamente in una query SQL senza sanificazione. Un attaccante può manipolare la query per estrarre dati, bypassare autenticazioni o cancellare tabelle. La contromisura corretta sono i prepared statements, che separano il codice SQL dai dati.

Cross-Site Scripting (XSS): consente di iniettare script JavaScript nelle pagine viste da altri utenti. Se un'applicazione mostra input dell'utente senza fare l'escape, chiunque può inserire codice che verrà eseguito nel browser delle vittime. La contromisura è fare sempre l'escape dell'output con funzioni come htmlspecialchars().

Una parte del lavoro è stata fare un security audit del codice del gruppo: ho trovato problemi reali come hash delle password esposti nelle risposte API, gestione degli errori che rivelava la struttura del database, e routing non protetto che permetteva accessi non autorizzati. Documentare ogni problema con un payload di test e riscrivere la parte corretta è stato più formativo che scriverlo da zero.

La cosa che mi ha interessato di più è stata ragionare non solo come sviluppatore, ma anche come un attaccante che cerca di sfruttare quelle vulnerabilità: è un cambio di prospettiva che cambia completamente il modo in cui si scrive codice.
