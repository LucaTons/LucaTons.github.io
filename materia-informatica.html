Le applicazioni Web

L'argomento che mi ha colpito di più durante l'anno è stato lo sviluppo delle applicazioni web. Abbiamo costruito un backend completo usando PHP come linguaggio lato server e MySQL per i dati, implementando le operazioni CRUD:

  • Create: aggiunge nuovi dati tramite il metodo POST
  • Read: legge e recupera i dati tramite GET
  • Update: modifica i dati esistenti tramite PUT
  • Delete: elimina i dati tramite DELETE

Abbiamo usato il PHP per il backend, cioè della logica del server e della gestione dei dati, mentre HTML lo abbiamo usato come frontend per testare le rotte create dal backend, inviando richieste e verificando le risposte. Oltre a creare il backend e il frontend, mi sono occupato personalmente è stata testare la sicurezza del sito, analizzando il codice del gruppo cercando vulnerabilità reali. Ne ho trovate diverse: hash delle password esposti nelle risposte API, gestione degli errori che rivelava la struttura del database e routing non protetto che permetteva accessi non autorizzati.

Test che ho svolto per testare la sicurezza

Le due vulnerabilità principali che ho testato sono state:

  • SQL Injection: si verifica quando l'input dell'utente viene inserito direttamente in una query SQL senza controlli. Un attaccante può manipolare la query per estrarre dati o bypassare l'autenticazione. La contromisura corretta sono i prepared statements, che separano il codice SQL dai dati.

  • Cross-Site Scripting (XSS): permette di iniettare codice JavaScript nelle pagine viste dagli altri utenti, che viene eseguito nel loro browser senza che se ne accorgano. La contromisura è fare sempre l'escape dell'output con funzioni come htmlspecialchars().

La cosa che mi ha interessato di più è stata ragionare non solo come sviluppatore ma anche come un attaccante che cerca di sfruttare quelle vulnerabilità. È un cambio di prospettiva che cambia completamente il modo in cui si scrive codice.
